# Правила работы с проектом my_lending

## Язык

- Всегда пиши на русском языке
- Комментарии в коде — на русском языке
- Названия переменных и функций — на английском (camelCase)

## Дизайн

- Не трогай дизайн без спроса — всегда спрашивай перед изменением визуальной части
- Любые изменения в UI/UX сначала согласовывай с Виктором

## Стиль работы

- Двигаемся последовательно, шаг за шагом
- Каждый шаг согласовываем перед выполнением
- Не усложняй — решения должны быть простыми и понятными
- Объясняй свои решения простым языком
- Делай сам всё что можешь — просить Виктора только если невозможно иначе
- Переход к следующему шагу — только с подтверждения Виктора

## Проектная документация

- Перед началом работы прочитай project.md — там описание проекта
- Перед началом задачи сверься с PLAN.md — там актуальный план и статус задач
- После завершения задачи — обнови PLAN.md (отметь выполненное)

## Прототип

- Лендинг строится на основе исследования лучших практик (research.md)
- Стиль и структура согласовываются с Виктором до начала вёрстки

## Безопасность

- Никогда не хардкодь пароли, токены, ключи API и секреты в код
- Используй переменные окружения (.env) для конфиденциальных данных
- Файл .env всегда должен быть в .gitignore

## Git

- Коммить каждую завершённую задачу отдельно с понятным описанием на русском
- Не коммить файлы с секретами (.env, ключи, токены)
- Перед коммитом проверь git status — не попало ли лишнее

## Технические правила

- Код должен быть чистым и читаемым
- Не добавляй зависимости без согласования
- Не удаляй и не перемещай существующие файлы без спроса
- Предлагай самое простое решение из возможных
- Все тексты интерфейса — на русском, без англицизмов

## Обучение

- Виктор проходит обучение по курсу "Вайбкодинг 2.0" (Нейроуниверситет)
- Все действия по проекту должны соответствовать методологии обучения
- Не переходим к следующему этапу, пока не выполнен текущий
